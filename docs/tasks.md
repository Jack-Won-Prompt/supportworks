# AI Agent 구현 작업 단위 리스트 (Tasks)

> **목적**: AI Agent 메뉴 전체 구현을 Claude Code가 한 번에 처리 가능한 작업 단위로 분해한 마스터 리스트
> **사용법**: 각 작업(T01, T02 ...)을 순서대로 Claude Code에게 요청. 완료 시마다 체크박스 표시 및 동작 확인
> **선행 조건**: `01_supportworks_env_checklist.md` 작성 완료, `ai-agent-spec.md` 저장 완료

---

## 사용 방법

### 작업 요청 표준 형식
```
@docs/ai-agent-spec.md
@docs/supportworks-env.md

[T01] 데이터 모델 마이그레이션 SQL 작성

요구사항:
- spec.md 섹션 4의 데이터 모델 기준
- env.md의 DB 종류에 맞는 마이그레이션 작성
- 인덱스, 외래키 제약조건 포함
- 마이그레이션 파일을 [경로]에 작성
```

### 작업 완료 확인
각 작업이 끝나면:
1. ✅ 코드 동작 확인
2. ✅ 컨벤션 준수 확인
3. ✅ 체크박스 표시
4. 다음 작업으로 진행

## 작업 진행 팁

### 1. 작업당 1인수 원칙
한 작업(T01 등)을 Claude Code와 한 세션에서 끝내는 것이 이상적

### 2. 완료 정의(Definition of Done)
- ✅ 코드 작성 완료
- ✅ 빌드/실행 성공
- ✅ 핵심 기능 자동 검증
- ✅ 컨벤션 준수 (Linter 통과)
- ✅ 다음 작업이 사용할 수 있는 상태

### 3. 매 작업 후 커밋
각 T작업 완료 시 Git 커밋. 커밋 메시지에 작업 ID 포함:
```
feat(ai-agent): [T01] add data model migrations
feat(ai-agent): [T02] add ORM entity classes
```

### 4. 막힐 때
작업 진행 중 spec.md에 명시되지 않은 결정 사항이 나오면:
- spec.md를 먼저 보완
- 환경 정보 체크리스트 보완
- 그 후 작업 진행

### 5. 컨텍스트 리셋
Claude Code 세션이 길어져 컨텍스트가 흐려지면:
- 새 세션 시작
- spec.md, env.md, tasks.md 다시 컨텍스트로 제공
- 현재 진행 중인 작업 ID 명시
